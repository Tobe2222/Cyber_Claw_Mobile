## 2026-07-22 10:03 (channel: #cyber-dev)
- **User:** Tobe reported "dont see anything on github yet" (likely GitHub mobile app cache on the releases list — v3.10.66 was published 2026-07-21 20:50 UTC) and two real bugs in the v3.10.66 active enrollment: "no notification after manual stop" and "it also did not stop after 30s as it says it will."
- **You:** v3.10.67 fixes both. Root cause: the v3.10.66 native thread's `finally` block released the AudioRecord but never called `EnrollmentCoordinator.end()` on the auto-stop path — so the BG/foreground mic gate stayed engaged until a manual stop. Also added a JS-side `setTimeout` watchdog (33s) as belt-and-suspenders in case the native thread ever gets stuck. For the no-notification bug: added a `NativeEventEmitter` event `activeEnrollmentStopped` that fires on both auto-stop and manual stop, with a `reason` arg. JS listener shows a Toast via `NativeBackground.showToast()` ("✅ Listening complete — review and lock profile" for auto, "⏹ Stopped" for manual) and cleans up timers/UI. Bumped to v3.10.67 (versionCode 293). Pushed branch + tag to origin, CI is building now. See CHANGES_3.10.67.md for the breakdown and the lessons "pair every native background action with a JS-visible signal" + "cleanup paths need to mirror success paths" + "a JS watchdog isn't a substitute for fixing the native side but it's still worth having".
