## 2026-07-22 14:30 (channel: #cyber-dev)
- **User:** Tobe reported "i noticed the chat still gives double texts. We can keep the one which states where it comes from" — screenshot showed the same typed message appearing twice in chat: once plain ("Hey my bro!") and once with `[From: Android Phone]` prefix.
- **You:** v3.10.68 fixes the dedupe. Root cause: the mobile appends the local typed message with plain text (no prefix), the desktop echoes it back with `[From: <deviceName>]` prefix added in `sync-server.js:320`. The v3.10.42 normalize only stripped leading emoji/non-word chars — it kept `[` and everything after, so bracketed prefixes like `[From: Android Phone]` survived on the echo side and the plain local text never matched. Two complementary fixes: (1) the mobile now prepends `[From: <deviceName>]` locally so both bubbles have identical text and dedupe works (the prefixed one stays because it's added first). The device name is cached from `auth_result.name` in SyncClient (`_deviceName` + `getDeviceName()`), defaulting to `'Android Phone'` to match the hardcoded `pair()` call. (2) `normalize()` also strips a leading `[From: ...]` block as defensive cleanup for old persisted chat history. Bumped to v3.10.68 (versionCode 294). Pushed branch + tag to origin, CI is building now. See CHANGES_3.10.68.md for the breakdown and the lesson "wire-format prefixes belong on the producer side — both sides should use the same prefix so dedupe works without special normalize gymnastics".
