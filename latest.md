- **You:** v3.10.80 ships a manual keyboard-avoidance fix for Android. v3.10.71 disabled KeyboardAvoidingView on Android and relied on windowSoftInputMode="adjustResize" to push the chat input up. That worked on Android <15, but the app's targetSdk is 36 — Android 15+ enforces edge-to-edge and silently DISABLES adjustResize for edge-to-edge apps (window doesn't resize, input stays at the bottom of the unresized window, keyboard covers it). Fix: track keyboard height via Keyboard.addListener('keyboardDidShow', ...) and apply it as paddingBottom on the inputContainer on Android. iOS keeps using KeyboardAvoidingView with behavior='padding' (the iOS-native path doesn't have this problem). Bumped to v3.10.80 (versionCode 304), tagged + pushed. See CHANGES_3.10.80.md for the breakdown and the lessons "don't trust 'the OS handles this' without testing on the target platform version" + "Android 15+ edge-to-edge enforcement is invisible until you trip over it — audit all native-handled behaviors when targetSdk changes" + "paddingBottom on the inputContainer is a self-contained keyboard-avoidance primitive that survives any OS-level window management change".
